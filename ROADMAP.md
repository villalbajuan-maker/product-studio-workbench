# Roadmap

This roadmap is intentionally practical.

The goal is to move from thesis to paid validation without overbuilding.

## Phase 0: Workbench Setup

Status: Done

- Create repository.
- Capture thesis.
- Define studio model.
- Define first commercial offer.
- Add landing app.
- Add validation docs.

## Phase 1: Commercial Validation Surface

Status: In progress

Goal:

> Make the project understandable and sellable enough to start real buyer conversations.

Work:

- Deploy landing page to Vercel.
- Review visual/copy quality.
- Make the sample decision map more compelling.
- Add contact capture or booking flow.
- Prepare outreach list.

Exit criteria:

- Landing is live.
- Sample artifact is visible.
- 20 prospects are listed.
- James can confidently explain the offer.

## Phase 2: Market Conversations

Status: Next

Goal:

> Test whether the pain, language, and offer resonate with qualified buyers.

Work:

- Start 10 buyer conversations.
- Capture notes using the call template.
- Track exact buyer language.
- Identify objections.
- Identify one real ambiguous opportunity per qualified prospect.

Exit criteria:

- 10 conversations completed.
- At least 3 prospects have a real case.
- At least 3 pilot offers made.

## Phase 3: Paid Pilot

Status: Next

Goal:

> Sell and deliver one paid Pre-Delivery Decision Map.

Work:

- Define pilot terms.
- Process one real opportunity.
- Deliver the decision map.
- Capture before/after value.
- Ask for testimonial, referral, or second opportunity.

Exit criteria:

- 1 paid pilot sold.
- 1 decision map delivered.
- Client confirms the output improved a pre-delivery decision.

## Phase 4: Pattern Recognition

Status: Later

Goal:

> Learn whether this is a repeatable business, a product opportunity, or a narrow service.

Work:

- Compare multiple pilots.
- Identify repeated inputs and outputs.
- Identify repeated buyer objections.
- Identify repeated parts of the artifact.
- Decide what should become productized.

Exit criteria:

- Clear pattern across pilots.
- Evidence of willingness to pay.
- Decision on next commercial package or product prototype.

## Phase 5: Studio System

Status: Later

Goal:

> Build the internal operating system only after commercial signal exists.

Work:

- Define SOPs.
- Define delivery process.
- Define reusable artifact system.
- Define product opportunity pipeline.
- Define partnership criteria in practice.

Exit criteria:

- The studio has repeatable paid work or product evidence.
